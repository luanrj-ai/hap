主要发现：

3 · 方法 & Caveats

  • 合成 fixtures：本人手写 30 份，刻意模仿真简历样式（真公司名、具体数字、不齐的 bullet）。仍是人写而非候选人提供。
  • 真人 fixtures：从 GitHub 公开 Profile README 抓取（如 github.com/swyxio/swyxio 等 22 个候选用户的 README），保留 19 个 ≥500 字符的。**这是真实人类撰写的内容**，但格式是 bio 不是简历——这就是 jd-match 数字差异的原因。
  • AI fixtures：同一批 30 份 GPT-5.5 生成的 AI 简历（naive 10 + coached 10 + adversarial 10）。两个 eval 用相同 AI 集。
  • 统计意义：n=19 真人 + n=30 AI 仍是小样本。结果方向对，绝对值再跑 ±5pp 浮动。
  • 未覆盖：data-ml 真人样本只 1 个（rasbt）——Data JD 上的真人数字代表性弱。Payments 真人样本 0 个。这些方向需要补样本。

4 · 给投资人 / 客户的对外说法

可以说：
  • "AI 简历检测 F1 94%"（两个数据集平均）
  • "Adversarial AI 抓获率 60–80%"（被指示绕过检测的 AI）
  • "真人非英语母语样本零误报"（在合成数据上 0 FP）

不能说：
  • "在真实候选人简历上的 F1" —— 我们还没有真候选人简历，只有 GitHub 公开 bio
  • "jd-match 提升 50%+" —— 这是合成数据上的数字，真人 bio 上提升只 +2pp
